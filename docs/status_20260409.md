- Rock detection: 80%+ confidence

### 2. Calibration ✅
- Two calibration sets: cropped (test) vs RTSP (live)

### 3. Game State Machine ✅
- States: IDLE → DELIVERY_IN_PROGRESS → ROCK_IN_FLIGHT → ROCKS_SETTLING → THROW_COMPLETE
- Rock tracking with velocity estimation
- Delivery detection via class presence

### 4. Scoring Logic ✅
- Calculates rocks in house
- Determines scoring team
- Counts points correctly

### 5. Unified Processor ✅
- Processes both near/far cameras
- House state calculation integrated
- 0.38 fps dual-camera on Pi

## Performance Notes

- Pi 4GB can handle REST API calls but not local inference pipeline
- Dual-camera processing: 0.38 fps
- Single-camera processing: 0.5-0.6 fps
- Acceptable for curling (slow game, ~1 throw per 30 seconds)

## Recommended Production Setup

1. **Option A: Keep Pi + REST API**
   - Process at 1-2 fps per camera
   - Sequential processing (near, then far, then wide)
   - Total latency: 1-2 seconds per frame

2. **Option B: Dedicated inference box**
   - Nvidia Jetson or similar
   - Local inference at 10+ fps
   - Real-time processing

3. **Option C: Hybrid**
   - Pi collects frames and coordinates
   - Sends to cloud/server for inference
   - Best for multiple sheets
