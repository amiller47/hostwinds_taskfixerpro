### Phase 2: Game State Machine ✅ (In Progress)

**Design (state_machine_design.md)**
- States: IDLE → DELIVERY_IN_PROGRESS → ROCK_IN_FLIGHT → ROCKS_SETTLING → THROW_COMPLETE → END_COMPLETE
- Rock tracking with velocity estimation
- Delivery detection via class presence
- Score calculation based on rock positions

**Implementation (game_tracker.py)**
- `RockTracker`: Tracks rocks across frames with velocity
- `DeliveryTracker`: Detects delivery start/end
- `GameTracker`: Main state machine with transitions

**Test Results (near_crop, 50 frames)**
- State transitions detected: idle → delivery_in_progress → rock_in_flight → idle
- Delivery detection works (curler class detected)
- Rock tracking detects rocks but velocity estimation needs refinement
